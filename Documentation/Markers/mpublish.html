Publishing project

The Publish Project component contains data necessary to present your project online. This component is attached to the object Rig in your scene.

  • Title: the title of the project.
  • Author: the creator(s) of the project.
  • Id: the ID of the project (by default it show the date and time of the projects creation). You can write your own or create new ID by clicking on New ID button. Please note the this ID is used to differentiate between projects by the servers.
  • Server IP: the IP address of the server.
  • Server Port: the Port of the server used for updates during the play mode. Alternatively, you can pick colors in Color Steps (if you use this mode, the text-based steps won’t be read)
  • Alt Port: this Port is used for email request and connections outside the play mode.
  • Email: the email to which the project status (surveys) is sent.
  • Description: project’s description.
  • New ID: generates a new ID reflecting the date and time.
  • Token: a token given to you by the server's manager. This token is necessary for the server to register your project and properly send you the survey results of online projects.
  • Register: registers the project ID and email with the server, using the token.
  • Final Publish: creates a "bundle" from the project that can be opened in another computer via the Folly Hub software. After creating the bundle, it will ask you for a destination where it copies and archives the data in a ZIP file named after the project’s ID. This file should be extracted as is (not in a new folder) in the Projects folder of Folly Hub on your or another PC.
